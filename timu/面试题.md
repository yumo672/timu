# 智能采购审批系统

框架：django

前端：html css js(简易页面即可，不使用django模板引擎)

数据库：Mysql

## 一、核心角色

1. **申请人**：提交采购申请、查看本人申请进度、接收打回理由并修改
2. **财务专员**：审批对应金额段申请、可打回申请并填写理由
3. **导师**：审批高金额申请、**查看全量申请及状态**、可打回申请并填写理由

## 二、审批规则

1. 金额＜1000 元：**系统自动通过**，无需人工审批
2. 1000 元≤金额＜5000 元：**财务专员审批**
3. 金额≥5000 元：**财务专员先审批 → 导师再审批**（双审批）

## 三、申请状态

待提交、待审批、审批中、已通过、已打回

## 四、核心功能

1. 申请人：新增 / 编辑采购申请（物品、数量、金额、用途）、查询本人申请状态
2. 审批人（财务 / 导师）：查看待审批列表、通过申请、**打回申请（必填理由）**
3. 导师：查看所有申请的完整状态列表
4. 通用：状态实时更新、打回理由展示

## 五、约束条件

1. 打回申请必须填写理由，不允许无理由打回
2. 多级审批按顺序执行，不可越级
3. 仅本人可查看 / 操作自己的申请，审批人仅能操作权限内申请

## 六、检查点

### 1. gitee仓库检查点

1. 已创建 gitee 仓库，并设置清晰的仓库名称与简介
2. 本地项目已成功关联远程仓库
3. 代码已至少完成一次提交并成功推送到 gitee
4. 仓库目录结构完整，包含前端、后端相关文件
5. 仓库内包含 `README`，说明项目名称、技术栈、启动方式
6. 仓库内包含必要的依赖文件，如 `requirements.txt`

### 2. 功能检查点

1. 申请人可以新增采购申请
2. 申请人可以编辑自己的采购申请
3. 申请人可以查看本人申请状态与审批进度
4. 金额低于 1000 元的申请可自动通过
5. 金额在 1000 元到 5000 元之间的申请会进入财务审批流程
6. 金额大于等于 5000 元的申请会进入财务和导师双审批流程
7. 审批人可以通过申请
8. 审批人可以打回申请，且打回时必须填写理由
9. 申请人可以查看打回原因并修改后重新提交
10. 导师可以查看所有申请及其状态
11. 不同角色只能看到并操作自己权限范围内的数据
12. 申请状态流转正确，如待提交、待审批、审批中、已通过、已打回
